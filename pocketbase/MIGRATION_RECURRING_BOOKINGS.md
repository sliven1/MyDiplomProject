# Миграция: Постоянное расписание (Recurring Bookings)

## Дата: 2026-02-27

## Что добавляем

Функционал постоянного расписания - ученик может забронировать занятия "Постоянно (каждую неделю)" одним запросом.

## Изменения в коде

✅ **Модель ScheduleSlot** (lib/models/schedule_slot.dart):
- Добавлено поле `isRecurring` (bool)
- Добавлено поле `recurringGroupId` (String?)

✅ **Сервис ScheduleService** (lib/service/schedule_service.dart):
- Добавлен метод `bookRecurringSlots()` - бронирует все доступные слоты на 3 месяца
- Добавлен метод `cancelRecurringGroup()` - отменяет всю группу одним запросом

✅ **UI** (lib/pages/tutor_schedule_view_page.dart):
- Чекбокс "Постоянно (каждую неделю)" в диалоге бронирования

## Изменения в PocketBase

### Шаг 1: Открыть PocketBase Admin UI

```bash
# Запустить PocketBase (если ещё не запущен)
cd pocketbase
docker-compose up -d

# Открыть в браузере
open http://localhost:8090/_/
```

### Шаг 2: Добавить поля в коллекцию `slots`

1. Перейти в **Collections** → **slots**
2. Нажать **+ New field**

#### Поле 1: `isRecurring`

- **Type**: `Bool`
- **Name**: `isRecurring`
- **Required**: ❌ (unchecked) - когда unchecked, по умолчанию будет false

Примечание: В PocketBase для Bool полей нет опции "default". Если поле не required, то по умолчанию оно будет `false`.

#### Поле 2: `recurringGroupId`

- **Type**: `Text`
- **Name**: `recurringGroupId`
- **Label**: Recurring Group ID
- **Pattern**: (оставить пустым)
- **Min length**: (оставить пустым)
- **Max length**: 100
- **Required**: ❌ (unchecked)
- **Description**: ID группы повторяющихся занятий

#### Поле 3: `bookingStatus` (если ещё не создано)

- **Type**: `Select`
- **Name**: `bookingStatus`
- **Max select**: 1 (Single)
- **Values** (добавь каждое значение отдельной строкой):
  - `free`
  - `pending`
  - `confirmed`
- **Required**: ☑️ (checked)

Примечание: В Select полях первое значение в списке автоматически становится default, поэтому `free` должен быть первым в списке.

### Шаг 3: Добавить индекс для `recurringGroupId`

1. В настройках коллекции `slots` перейти в **Indexes**
2. Нажать **+ New index**
3. Добавить индекс:
   - **Name**: `idx_recurringGroupId`
   - **Columns**: `recurringGroupId`
   - **Unique**: ❌ (unchecked)

### Шаг 4: Сохранить изменения

Нажать **Save changes** в правом верхнем углу

## Как это работает

### Сценарий 1: Бронирование постоянного расписания

1. Ученик открывает расписание репетитора
2. Выбирает слот (например: Понедельник 10:00-11:00, 2 марта)
3. Нажимает "Забронировать"
4. В диалоге включает чекбокс "Постоянно (каждую неделю)"
5. Система:
   - Генерирует уникальный `recurringGroupId` (например: `recurring_1709123456_abc12345`)
   - Находит все свободные слоты на Понедельник 10:00-11:00 на ближайшие 3 месяца
   - Бронирует их все с одинаковым `recurringGroupId` и `isRecurring=true`
   - Показывает: "✅ Постоянное расписание создано! Забронировано 12 занятий"

### Сценарий 2: Отмена постоянного расписания

1. Ученик/репетитор отменяет постоянное расписание
2. Система находит все слоты с этим `recurringGroupId`
3. Отменяет их все одним запросом
4. Показывает: "✅ Отменено 12 занятий"

### Сценарий 3: Автоматическое бронирование новых слотов

1. Репетитор создаёт новые слоты через Weekly Template (на месяц вперёд)
2. Система проверяет активные постоянные расписания (`recurringGroupId`)
3. Автоматически бронирует новые слоты для учеников с постоянным расписанием
